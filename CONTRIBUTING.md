# Contributing to Tinnitus App

Thank you for contributing. Please read these guidelines before opening issues or pull requests.

## Branching

- `main` / `master` — production-ready code, protected
- Feature branches: `feature/<short-description>`
- Bug fixes: `fix/<short-description>`

## Pull requests

- Keep PRs focused — one concern per PR
- Fill in the PR template fully
- Request at least one review before merging
- Squash commits on merge

## Commit messages

Follow [Conventional Commits](https://www.conventionalcommits.org/):

```
feat: add onboarding questionnaire
fix: correct audio session handling on iOS
chore: bump flutter to 3.x
```

## Secrets

Never commit API keys, tokens, or credentials. Use environment variables or a secrets manager.

## Questions

Open a GitHub Discussion or reach out to the maintainer directly.
